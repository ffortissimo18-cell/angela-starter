# Урок 5. Google Календарь 📅

**Что добавляем:** бот видит твои встречи на сегодня (показывает их утром) и
может создавать события по просьбе — «поставь созвон завтра в 15:00».

Этот модуль использует ту же настройку Google, что и почта (Урок 4).

---

## Если Google уже настроен (прошла Урок 4)

Совсем коротко:

1. В Google Cloud включи **Google Calendar API**
   (**APIs & Services → Library → Google Calendar API → Enable**), если не
   включала раньше.
2. Railway → **Variables** → добавь:
   ```
   ENABLE_GCAL=true
   ```
3. **Авторизуйся заново** — у бота появилось новое право (календарь), его нужно
   подтвердить. Открой `https://your-app.up.railway.app/google/auth`, разреши
   доступ ещё раз. Готово.

## Если Google ещё не настраивала

Сначала пройди **Часть А** из [Урока 4](04-add-gmail.md) (создать проект,
экран согласия, OAuth Client ID), но на шаге «включи API» включи **Google
Calendar API**. Потом вернись сюда и сделай шаги 2–3 выше.

---

## Проверь

- Спроси: «что у меня сегодня по календарю?» — покажет встречи.
- Скажи: «поставь встречу завтра в 11:00 на полчаса, назови „созвон с Аней“» —
  создаст событие. Можешь добавить ссылку (например, на Zoom) — бот положит её в
  место встречи, а участникам Google разошлёт приглашения.

## Если не работает

- **Видит почту, но не календарь** → ты не прошла повторную авторизацию после
  включения `ENABLE_GCAL`. Открой `/google/auth` ещё раз (см. шаг 3).
- **Ошибка про API** → не включён Google Calendar API в Google Cloud.

> Бот работает с твоим основным календарём (primary). Хочешь усложнить — это уже
> в духе [Урока 7](07-build-your-own-tool.md).
