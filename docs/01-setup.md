# Урок 1. Запуск бота за ~30 минут

**Что строим:** живого бота в Telegram, который утром и вечером пишет тебе
сам, а в любой момент отвечает на сообщения.

**Не нужно уметь программировать.** Нужно по шагам получить 5 ключей и вставить
их в нужные поля. Делай по порядку, не пропускай.

---

## Шаг 0. Скопируй проект себе на GitHub

1. Заведи аккаунт на [github.com](https://github.com), если его нет.
2. Открой страницу этого проекта и нажми **Fork** (вверху справа) — появится
   твоя личная копия. Все дальнейшие шаги — с ней.

---

## Шаг 1. Создай Telegram-бота (5 мин)

1. В Telegram найди **@BotFather** и напиши `/newbot`.
2. Придумай имя (например, «Моя Анжела») и username (должен заканчиваться на
   `bot`, например `my_angela_bot`).
3. BotFather пришлёт **токен** — длинную строку вида `7123456:AAH...`.
   Скопируй её. Это `TELEGRAM_BOT_TOKEN`.

## Шаг 2. Узнай свой chat_id (1 мин)

Бот должен отвечать только тебе. Для этого нужен твой ID.

1. Найди в Telegram **@userinfobot** и напиши ему `/start`.
2. Он ответит числом `Id: 123456789` — это твой `TELEGRAM_CHAT_ID`.

## Шаг 3. Получи ключ Claude (5 мин)

1. Зайди на [console.anthropic.com](https://console.anthropic.com), зарегистрируйся.
2. **Plans & Billing** → пополни баланс (хватит $5 для старта).
3. **API Keys** → **Create Key** → скопируй. Это `ANTHROPIC_API_KEY`
   (показывается один раз — сохрани сразу).

## Шаг 4. Создай базу данных Supabase (10 мин)

1. Зайди на [supabase.com](https://supabase.com), **New project**. Задай имя и
   пароль (пароль просто сохрани, дальше не нужен). Подожди пару минут, пока
   проект создаётся.
2. Слева открой **SQL Editor** → **New query**.
3. Открой файл `schema.sql` из проекта, скопируй **всё** содержимое, вставь в
   редактор и нажми **Run**. Должно написать «Success». Это создало таблицы.
4. Слева **Project Settings → API**. Скопируй два значения:
   - **Project URL** → это `SUPABASE_URL`
   - **Project API keys → `service_role`** (нажми «reveal») → это `SUPABASE_KEY`

   ⚠️ `service_role` — секретный ключ. Он живёт только на сервере (Railway),
   никому его не показывай и не выкладывай на GitHub.

## Шаг 5. Запусти на Railway (10 мин)

Railway — это где бот будет работать круглосуточно.

1. Зайди на [railway.app](https://railway.app) через GitHub.
2. **New Project → Deploy from GitHub repo** → выбери свой форк.
3. Railway начнёт сборку. Пока идёт — открой вкладку **Variables** и добавь
   переменные (кнопка **New Variable**), по одной:

   | Имя | Значение |
   |-----|----------|
   | `TELEGRAM_BOT_TOKEN` | токен из шага 1 |
   | `TELEGRAM_CHAT_ID` | число из шага 2 |
   | `ANTHROPIC_API_KEY` | ключ из шага 3 |
   | `SUPABASE_URL` | URL из шага 4 |
   | `SUPABASE_KEY` | service_role из шага 4 |
   | `TIMEZONE` | твой пояс, напр. `Europe/Kyiv` |

4. После добавления переменных Railway пересоберётся сам. Открой вкладку
   **Deploy Logs** — когда увидишь строку `бот запущен, слушаю сообщения…`,
   всё готово.

## Шаг 6. Проверь

1. Открой своего бота в Telegram (по username из шага 1), нажми **/start**.
2. Напиши ему что-нибудь, например: «привет». Должен ответить.
3. Утром и вечером (в часы `MORNING_HOUR` / `EVENING_HOUR` твоего пояса) он
   напишет первым. Не хочешь ждать — поставь `MORNING_HOUR` на ближайший час в
   Variables и перезапусти: бот «догонит» пропущенный чекин на старте.

🎉 **Готово!** У тебя работает личный ассистент.

---

## Если что-то не так

- **Бот молчит** → проверь, что `TELEGRAM_CHAT_ID` — это именно твой ID, и что в
  логах Railway нет ошибки про переменные.
- **`Не заданы обязательные переменные`** в логах → какую-то из 5 переменных не
  добавила или опечатка в имени. Имена должны быть точь-в-точь.
- **Ошибка про billing/credit balance** → пополни баланс на console.anthropic.com.
- **Хочешь посмотреть, что происходит** → Railway → вкладка **Logs**.

## (Для технических) Запуск локально

Нужен Python 3.11+.

```bash
cp .env.example .env      # заполни значения в .env
pip install -r requirements.txt
python -m assistant
```
