# Урок 7. Сделай боту новый навык 🛠

Здесь ты добавишь боту свой собственный инструмент. Это уже немного
программирование — но по шаблону, который повторяется. Поймёшь один раз —
сможешь добавлять что угодно.

**Пример урока:** трекер воды. Бот будет считать выпитые стаканы за день, если
сказать «выпила стакан воды», и отвечать на «сколько я сегодня выпила?».

## Как устроен любой инструмент

Три части (посмотри готовый пример — `assistant/tools/core.py`):
- **`TOOLS`** — описание для Claude: как называется навык и какие у него поля
- **функция-обработчик** — что реально происходит (записать/прочитать в базе)
- **`HANDLERS`** — связывает имя навыка с функцией

Плюс по желанию `PROMPT_ADDON` — подсказка боту, когда этот навык применять.

---

## Шаг 1. Таблица в базе

Supabase → **SQL Editor** → выполни:

```sql
create table if not exists water_log (
  id          bigint generated by default as identity primary key,
  date        date        not null default current_date,
  glasses     int         not null default 1,
  created_at  timestamptz not null default now()
);
```

## Шаг 2. Файл инструмента

Создай файл `assistant/tools/water.py` со следующим содержимым:

```python
"""Трекер воды — пример своего инструмента."""

from datetime import date
from assistant.db import supabase

PROMPT_ADDON = (
    "Если человек говорит, что выпил воды («выпила стакан», «попила»), "
    "вызови log_water. На вопрос «сколько выпила» — water_today."
)

TOOLS = [
    {
        "name": "log_water",
        "description": "Записать выпитые стаканы воды (по умолчанию 1).",
        "input_schema": {
            "type": "object",
            "properties": {"glasses": {"type": "integer", "default": 1}},
        },
    },
    {
        "name": "water_today",
        "description": "Сколько стаканов воды выпито сегодня.",
        "input_schema": {"type": "object", "properties": {}},
    },
]


def _log_water(data: dict) -> dict:
    glasses = data.get("glasses", 1)
    supabase.table("water_log").insert({"glasses": glasses}).execute()
    return {"logged": glasses}


def _water_today(data: dict) -> dict:
    rows = (
        supabase.table("water_log")
        .select("glasses")
        .eq("date", date.today().isoformat())
        .execute()
        .data
    )
    return {"total": sum(r["glasses"] for r in rows)}


HANDLERS = {"log_water": _log_water, "water_today": _water_today}
```

## Шаг 3. Включатель в настройках

В `assistant/config.py` найди блок «Опциональные модули» и добавь строку:

```python
ENABLE_WATER = _flag("ENABLE_WATER")
```

## Шаг 4. Подключи в реестре

В `assistant/tools/registry.py`, рядом с другими модулями, добавь:

```python
    if config.ENABLE_WATER:
        from assistant.tools import water
        tools += water.TOOLS
        handlers.update(water.HANDLERS)
        addons.append(water.PROMPT_ADDON)
```

## Шаг 5. Включи и проверь

Railway → **Variables** → `ENABLE_WATER=true`. После пересборки скажи боту:
«выпила два стакана воды», потом «сколько я сегодня выпила?» — должен ответить
«2».
